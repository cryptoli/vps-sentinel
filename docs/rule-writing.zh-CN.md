# 规则编写指南

规则必须是防御型、可解释、有证据、默认安全的。不要加入攻击利用、爆破、第三方扫描或隐蔽控制能力。

## 原则

- 使用结构化证据，不要只靠单个字符串匹配。
- 同类规则应复用已有数据结构和评分器，避免另写一套逻辑。
- 区分配置风险、可疑行为和已确认入侵证据。
- 为正例和反例都添加测试，反例应覆盖包升级、正常运维、VPN/代理、容器和云初始化。
- 告警消息要给出影响和建议，不要只输出技术字段。

## 规则类型

- SSH/account：登录、爆破、密钥、用户和权限变化。
- File/persistence：关键文件、systemd、cron、shell profile 和持久化入口。
- Process/GPU：进程身份、父进程链、执行文件属性、CPU/GPU 行为和出站画像。
- Network/web：公网监听、Web 探查、攻击路径聚合、响应候选。
- Baseline：语义漂移评分和复核动作。
