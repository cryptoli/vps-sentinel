# 误报处理

`vps-sentinel` 的目标是尽早发现异常信号，而不是替代人工确认。遇到告警时应先阅读证据、规则、风险等级和处置建议，再决定是否隔离或刷新基线。

常见误报来源：

- 软件包升级导致 systemd unit、监听端口或关键文件变化。
- 管理脚本包含 WebShell 风格关键字，但实际属于合法运维。
- 反向代理、VPN、转发工具产生长期监听或高连接数。
- 新机器初始化、cloud-init、备份任务或自动运维修改账号、密钥或计划任务。

推荐流程：

1. 检查节点名称、时间、规则 ID、对象和证据字段。
2. 对照近期运维、包管理器日志、systemd unit owner、ExecStart 和文件 owner。
3. 确认是误报后，在面板中复核为 `false_positive`，或在本机刷新对应基线。
4. 不要为了消除单次噪音直接扩大 allowlist，应先确认是否属于稳定的正常行为。
